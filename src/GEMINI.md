# Projekt-Spezifische Anweisungen 🛠️

Dieses Projekt folgt dem **src-layout**. Bitte beachte die folgenden Konventionen:

## Namenskonventionen
- Alle Ordner und Dateien werden **kleingeschrieben**.
- Ordner nutzen Bindestriche (`-`), Dateien nutzen Unterstriche (`_`).

## Dokumentation
- Jede Funktion und Klasse **muss** einen Docstring haben.
- Nutze Navigation Tags für eine bessere Übersicht:
    - `!`: Wichtiger Hinweis
    - `SECTION:`: Logischer Abschnitt
    - `DEF:`: Funktionsdefinition
    - `ENTRY:`: Einstiegspunkt
    - `CONFIG:`: Konfiguration
    - `BRIDGE:`: Schnittstelle
    - `STATE:`: Zustandsverwaltung

## Kommentare
- Inline-Kommentare (`#`) dienen dazu, das **WARUM** zu erklären, nicht das **WAS**.
